## What is todo-harvest?

`todo-harvest` scans your codebase for technical debt markers (`TODO`, `FIXME`, `HACK`, `BUG`, `NOTE`, `DEPRECATED`) and produces structured reports showing what needs attention, who wrote it, and when.

```bash
npx todo-harvest scan ./src
```

It's the tool that finally makes your forgotten `// TODO: fix this before launch` comments visible — and actionable.

## The Problem

Every codebase accumulates comments like:

```javascript
// TODO: handle edge case when user is anonymous
// FIXME: this breaks when API rate limit is hit
// HACK: temporary workaround for payment bug
// NOTE: remove this before v2.0 launch
```

These comments are invisible in code review, forgotten in sprints, and never surface until production incidents. `todo-harvest` makes them a first-class artifact.

## CI Enforcement

The most powerful feature: fail your build when technical debt accumulates beyond policy.

```yaml
# .github/workflows/debt-check.yml
- name: Check technical debt
  run: npx todo-harvest scan ./src --fail-over 20
```

```bash
# Exit 1 if more than 20 TODOs
npx todo-harvest scan --fail-over 20

# Exit 1 if any FIXME comments exist
npx todo-harvest scan --fail-on-fixme
```

## Quick Start

```bash
# Scan current directory
npx todo-harvest scan

# Scan specific path, JSON output
npx todo-harvest scan ./src --format json

# With git blame attribution
npx todo-harvest scan ./src --blame

# CI enforcement: fail if over limit
npx todo-harvest scan ./src --fail-over 15
```
